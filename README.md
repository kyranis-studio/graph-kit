## 🚀 Quick Start

Here is a minimal example of creating a graph, registering a custom node, connecting data, and executing it.

```typescript
import { GraphKit } from 'jsr:@kyranis-studio/graph-kit';

// 1. Create a graph
const graph = GraphKit.createGraph({ name: 'Basic Math Pipeline' });

// 2. Register node types
graph.registerNodeType('add', {
  inputs: [
    { id: 'a', name: 'A', type: 'number', required: true },
    { id: 'b', name: 'B', type: 'number', required: true },
  ],
  outputs: [{ id: 'result', name: 'Result', type: 'number' }],
  execute: async (inputs) => ({ result: inputs.a + inputs.b }),
});

// 3. Add nodes
const n1 = graph.addNode('add', { data: { a: 5, b: 3 } });
const n2 = graph.addNode('add', { data: { b: 10 } });

// 4. Connect nodes via edges
graph.addEdge({
  sourceNodeId: n1.id,
  sourcePortId: 'result',
  targetNodeId: n2.id,
  targetPortId: 'a',
});

// 5. Execute the graph
const result = await graph.execute();
console.log(`Final Result: ${result.values.get(`${n2.id}.result`)}`); // Outputs: 18
```

## 🧠 Core Concepts

### Nodes
Nodes are the fundamental processing units. Each node has:
- `type`: A registered type that defines inputs, outputs, and the execution logic.
- `inputs` / `outputs`: Typed ports for receiving and sending data.
- `data`: Default configuration values merged with incoming edge data at runtime.
- `execute`: The async function that processes inputs and returns outputs.

### Ports
Ports define the interfaces for data flow:
- `type`: Strongly typed (`'string' | 'number' | 'boolean' | 'object' | 'array' | 'any'`).
- `required`: Whether a connection is mandatory for execution.
- `defaultValue`: Fallback data when no connected edge provides a value.

### Edges
Edges connect an output port of a source node to an input port of a target node, defining the execution topology.

## ⚙️ Execution & Debugging

GraphKit provides multiple execution engines to suit different environments, from headless production servers to interactive CLI applications.

### Sequential Execution (Default)
The `ExecutionEngine` executes nodes in topological order.

```typescript
import { ExecutionEngine } from 'jsr:@kyranis-studio/graph-kit';

// Log levels: 'silent' | 'minimal' | 'verbose'
const engine = new ExecutionEngine({ logLevel: 'minimal' });
const result = await engine.execute(graph);
```

### Interactive Debug Mode
The `DebugExecutionEngine` provides rich interactive step-through execution, pausing execution between nodes and rendering real-time streaming chunks to the console.

```typescript
import { DebugExecutionEngine } from 'jsr:@kyranis-studio/graph-kit';

const debugEngine = new DebugExecutionEngine({
  stepMode: true,          // Wait for SPACE key between nodes
  forceFullLog: true,      // Force verbose output
  forceStreaming: true,    // Force streaming mode on AI nodes
});

await debugEngine.execute(graph);
```

### Custom Loggers
You can extend the engine with custom formatters and loggers:

```typescript
import { ExecutionLogger, ConsoleLogger, DefaultFormatter } from 'jsr:@kyranis-studio/graph-kit';

class JsonFormatter {
  format(entry) {
    return JSON.stringify({ timestamp: entry.timestamp, level: entry.level, text: entry.text });
  }
}
```

## 🤖 AI Integration

GraphKit comes with pre-built nodes for major AI providers.

### Ollama, OpenAI, & OpenRouter

```typescript
import { GraphKit, registerOllamaNodes } from 'jsr:@kyranis-studio/graph-kit';

const graph = GraphKit.createGraph();
registerOllamaNodes(graph); // Or registerOpenAINodes / registerOpenRouterNodes

graph.addNode('ollama-chat', {
  data: {
    model: 'llama3.2',
    prompt: 'Explain node graphs briefly.',
    temperature: 0.7,
    streaming: true, // Native streaming support
  },
});

await graph.execute();
```

### Real-Time Streaming & Thinking
All providers support event-driven streaming, making it trivial to build reactive UIs or CLI spinners:

```typescript
graph.on('llmStreamChunk', ({ nodeId, state }) => {
  // state.thinking - DeepSeek/Claude reasoning content
  // state.response - Standard generated text
  if (state.done) {
    console.log(`\n[Stream complete]`);
  }
});
```

## 🔄 Workflows & Routing

The `Workflow` engine provides state-based execution with conditional routing, enabling complex loops and agentic logic that goes beyond a standard DAG (Directed Acyclic Graph).

```typescript
const workflow = graph.createWorkflow({
  startNode: 'agent-start',
  endNode: 'final-output',
});

workflow.connect('agent-start.output', 'processor.input');

// Conditional Routing based on execution state
workflow.addConditionalEdge({
  sourceNodeId: 'processor',
  condition: (state) => {
    const confidence = state.values.get('processor.confidence');
    return confidence > 0.8 ? 'final-output' : 'agent-start'; // Loop back if uncertain
  },
});

await workflow.run();
```

## 📚 API Reference

### `GraphKit`
- `GraphKit.createGraph(config?)` - Create a new graph instance.
- `GraphKit.fromJSON(json, nodeTypes?)` - Restore a graph from a serialized JSON string.

### `Graph`
- **Node Management**: `registerNodeType()`, `addNode()`, `removeNode()`, `updateNodeData()`, `getNode()`.
- **Edge Management**: `addEdge()`, `removeEdge()`, `getEdgesForNode()`, `getPredecessors()`, `getSuccessors()`.
- **Execution & Middleware**: `execute()`, `createWorkflow()`, `use()`.
- **Events**: `on()`, `off()`, `emit()`.
- **Export**: `toJSON()`, `toMermaid()`, `toDOT()`.

### Event System
Available graph events:
- `nodeStart`: Fired before a node executes.
- `nodeComplete`: Fired after successful node execution.
- `nodeError`: Fired when a node throws an error.
- `graphComplete`: Fired when the entire graph finishes.
- `llmStreamChunk`: Real-time streaming chunks from AI nodes.
