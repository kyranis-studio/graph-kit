# GraphKit - Node Graph Library for Deno 2

> **AI-Generated Code Notice**: This library is entirely generated by AI (opencode/BigPickle) as part of the Kyranis Studio project.

GraphKit is a lightweight, TypeScript-first library for building node graph applications in Deno 2. It provides the computational layer for creating, connecting, and executing node-based workflows with AI-first features.

## Features

- **Node System**: Typed units with inputs, outputs, and execution logic
- **Port System**: Type-safe data flow between nodes
- **Edge System**: Connect output ports to input ports
- **Execution Engine**: Three log levels (silent/minimal/verbose) with real-time LLM streaming
- **Debug Engine**: Interactive step-through execution with rich CLI feedback
- **Workflow Engine**: State-based execution with conditional edges
- **Log Engine**: Extensible logging with custom formatters and loggers
- **AI Providers**: Built-in support for Ollama, OpenAI, and OpenRouter with streaming and thinking
- **Function Calling**: Define tools/functions for LLMs to invoke
- **Visualization**: Export graphs to Mermaid and DOT formats
- **Middleware**: Koa-style middleware for execution hooks
- **TypeScript Native**: Full strict mode with comprehensive types

## Quick Start

```typescript
import { GraphKit } from './mod.ts';

// Create a graph
const graph = GraphKit.createGraph({ name: 'Basic Math' });

// Register node types
graph.registerNodeType('add', {
  inputs: [
    { id: 'a', name: 'A', type: 'number', required: true },
    { id: 'b', name: 'B', type: 'number', required: true },
  ],
  outputs: [{ id: 'result', name: 'Result', type: 'number' }],
  execute: async (inputs) => ({ result: inputs.a + inputs.b }),
});

// Add nodes and connect them
const n1 = graph.addNode('add', { data: { a: 5, b: 3 } });
const n2 = graph.addNode('add', { data: { b: 10 } });

graph.addEdge({
  sourceNodeId: n1.id,
  sourcePortId: 'result',
  targetNodeId: n2.id,
  targetPortId: 'a',
});

// Execute
const result = await graph.execute();
console.log(`Result: ${result.values.get(`${n2.id}.result`)}`); // 18
```

## Core Concepts

### Nodes

Nodes are the fundamental processing units. Each node has:
- `type`: A registered type that defines inputs, outputs, and executor
- `inputs`: Typed ports for receiving data
- `outputs`: Typed ports for sending data
- `data`: Configuration values merged with edge inputs at execution time
- `execute`: Async function that processes inputs and returns outputs

```typescript
graph.registerNodeType('greet', {
  inputs: [
    { id: 'name', name: 'Name', type: 'string', required: true },
    { id: 'greeting', name: 'Greeting', type: 'string', required: false, defaultValue: 'Hello' },
  ],
  outputs: [{ id: 'message', name: 'Message', type: 'string' }],
  execute: async (inputs) => ({
    message: `${inputs.greeting}, ${inputs.name}!`,
  }),
});
```

### Ports

Ports define the typed interfaces for data flow:
- `id`: Unique identifier within the node
- `name`: Human-readable label
- `type`: `'string' | 'number' | 'boolean' | 'object' | 'array' | 'any'`
- `required`: Whether a connection is mandatory
- `defaultValue`: Fallback when no edge provides data

### Edges

Edges connect output ports to input ports, defining the data flow:
```typescript
graph.addEdge({
  sourceNodeId: node1.id,
  sourcePortId: 'result',
  targetNodeId: node2.id,
  targetPortId: 'input',
});
```

## Log Engine

GraphKit includes a flexible log engine with three built-in levels:

```typescript

// Or via graph.execute()
```

## Execution Modes

### Sequential (Default)

The `ExecutionEngine` executes nodes in topological order:

```typescript
const engine = new ExecutionEngine();
const result = await engine.execute(graph);

// With custom log level
const verboseEngine = new ExecutionEngine({ logLevel: 'verbose' });
await verboseEngine.execute(graph);
```

### Debug Mode

The `DebugExecutionEngine` provides interactive step-through execution:

```typescript
import { DebugExecutionEngine } from './mod.ts';

const debugEngine = new DebugExecutionEngine({
  stepMode: true,          // Wait for SPACE key between nodes
  forceFullLog: true,      // Force verbose output
  forceStreaming: true,    // Force streaming mode on chat nodes
  onNodeStart: (info) => console.log(`Starting: ${info.nodeId}`),
  onNodeComplete: (info) => console.log(`Done: ${info.nodeId}`),
  onNodeError: (info) => console.error(`Error: ${info.nodeId}`),
  onStreamChunk: (chunk) => console.log('Stream:', chunk.state.response.length, 'chars'),
});

const result = await debugEngine.execute(graph);
```

### Workflow Mode

The `Workflow` engine provides state-based execution with conditional edges:

```typescript
const workflow = graph.createWorkflow({
  startNode: 'agent1',
  endNode: 'final-output',
  onStateUpdate: (state) => console.log('State updated'),
  logLevel: 'minimal',  // 'silent' | 'minimal' | 'verbose'
});

workflow.connect('agent1.output', 'agent2.input');
workflow.connect('agent2.output', 'final-output.input');

// Add conditional routing
workflow.addConditionalEdge({
  sourceNodeId: 'router',
  condition: (state) => {
    const val = state.values.get('router.result');
    return val > 0 ? 'positive-path' : 'negative-path';
  },
});

const result = await workflow.run();
```

### Function Calling

```typescript
import { createOllamaProvider } from './ai/providers/ollama.ts';
import type { ToolDefinition, ChatMessage } from './ai/providers/types.ts';

const weatherTool: ToolDefinition = {
  type: 'function',
  function: {
    name: 'get_weather',
    description: 'Get weather for a location',
    parameters: {
      type: 'object',
      properties: {
        location: { type: 'string' },
      },
      required: ['location'],
    },
  },
};

const ollama = createOllamaProvider();
const messages: ChatMessage[] = [
  { role: 'user', content: 'Weather in Tokyo?' },
];

// Tool calling loop
let iterations = 0;
while (iterations < 5) {
  iterations++;
  const response = await ollama.chat({
    model: 'llama3.1',
    messages,
    tools: [weatherTool],
  });

  if (response.message.tool_calls) {
    messages.push(response.message);
    for (const tc of response.message.tool_calls) {
      const args = JSON.parse(tc.function.arguments);
      const result = await getWeather(args.location);
      messages.push({ role: 'tool', content: result, tool_call_id: tc.id });
    }
  } else {
    console.log(response.message.content);
    break;
  }
}
```

- `registerNodeType(type, definition)` - Register a node type
- `addNode(type, config?)` - Add a node instance
- `removeNode(nodeId)` - Remove a node and its edges
- `updateNodeData(nodeId, data)` - Merge data into a node
- `addEdge(config)` - Connect nodes
- `removeEdge(edgeId)` - Remove an edge
- `getNode(nodeId)` - Get a node by ID
- `getEdgesForNode(nodeId)` - Get all edges for a node
- `getPredecessors(nodeId)` - Get input nodes
- `getSuccessors(nodeId)` - Get output nodes
- `validate()` - Validate graph structure

- `toDOT()` - Export to DOT
- `createWorkflow(config)` - Create a workflow
- `use(middleware)` - Add middleware
- `on(event, handler)` - Subscribe to events
- `off(event, handler)` - Unsubscribe from events
